This is FreeBSD 15 configuration information for my IBM Thinkpad X1 Gen 9 laptop.

I don't really have a good place to put this, so I'm putting it here because I don't want it lost in case I ever need to reinstall FreeBSD. Maybe this will help someone else running on a Thinkpad.

# Packages
acpi_ibm

# Sound

This was a super bitch to get into a working state without pops, clicks and scratching. I don't think it's perfect yet, but it's Good Enough (TM).

### /boot/loader.conf

# Prevent the audio hardware from sleeping/throttling
hw.pci.do_power_nodriver=3

# Force the system to use MSI for audio processing
hw.hdac.msi=1

### /boot/device.hints

hint.hdaa.0.pwrstate="0"
hint.hdaa.0.nid19.config="as=1 seq=0"
hint.hdaa.0.nid20.config="as=1 seq=2"
hint.hdaa.0.nid21.config="as=1 seq=15"
hint.hdaa.0.nid23.config="as=1 seq=0 device=Speaker"
hint.hdaa.0.nid33.config="as=1 seq=15 device=Headphones"
hint.hdaa.0.nid24.config="as=1 seq=2"
hint.hdaa.0.nid26.config="as=1 seq=1"
hint.hdaa.0.nid27.config="as=1 seq=2"
hint.hdaa.0.nid30.config="as=1 seq=1"

### /etc/sysctl.conf

# Disable real-time channel state notifications to mixers
hw.snd.verbose=0

# Lock the mixer to standard hardware limits (stops volume over-amplification) - Various tinkering with this.
#hw.snd.vpc_0db=1
hw.snd.vpc_0db=75

# Raise the latency slightly to prevent the control panel from causing an underrun. Tried various values from 2 to 8 here.
hw.snd.latency=2

# Stop polling the hardware. 
dev.hdac.0.polling=0

# Smooth out the processing window to eliminate clicks. Tried various values here.
hw.snd.feeder_buffersize=4096
hw.snd.feeder_rate_quality=4

# Allow multiple applications to share the hardware channels natively. In theory, these didn't seem to help, but leaving them here just in case.
#hw.snd.maxautovchans=8
#dev.pcm.0.play.vchans=8
dev.pcm.0.play.vchanrate=48000

# Firefox

Go into about:config and set:

# Force Firefox to use your sound instead of creating its own channel. The problem I couldn't solve was that it kept setting the volumeto 100%, and resetting it back to 100% every time there was a commercial in youtube or wherever.
media.cubeb.backend = "oss"

# This was my original attempt to control the max volume issue above, but I'm not strictly sure it's necessary. But, I've left it for now.
media.volume_scale = 0.3

### Misc

# Generally did this any time I was fiddling with the audio settings before a reboot.
sudo rm /var/db/mixer*-state

# Tinkered with this to basically set the base volume level. Note the syntax, the intertubes will sometimes give you old syntax which doesn't work.

mixer pcm=0.75
mixer vol=1.00

### Things That Failed

cpu_microcode caused a freeze during startup. I had to learn what gpi was and how to get out of that mess.

hint.hdac.0.config="ovref100" - This was originally an attempt to solve quiet headphones, but it just caused scratchy sound. And the headphone issue was mainly solved by adjustable volume on the headphones themselves.

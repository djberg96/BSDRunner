I don't really have a good place to put this, so I'm putting it here because I
don't want it lost in case I ever need to reinstall FreeBSD. Maybe this will help
someone else running on a Thinkpad. Or maybe you have some suggestions for me.

## Sound

This was a super bitch to get into a working state without pops, clicks and scratching. I don't think it's perfect yet, but it's Good Enough (TM).

### /boot/loader.conf

# Avoid UEFI boot stuff
tpm_load="NO" 

# Enable extra support for hotkeys and other bits on Thinkpads
acpi_ibm_load="YES"

# Prevent the audio hardware from sleeping/throttling
hw.pci.do_power_nodriver=3

# Force the system to use MSI for audio processing
hw.hdac.msi=1

### /boot/device.hints

hint.hdaa.0.pwrstate="0"
hint.hdaa.0.nid19.config="as=1 seq=0"
hint.hdaa.0.nid20.config="as=1 seq=2"
hint.hdaa.0.nid21.config="as=1 seq=15"
hint.hdaa.0.nid23.config="as=1 seq=0 device=Speaker"
hint.hdaa.0.nid33.config="as=1 seq=15 device=Headphones"
hint.hdaa.0.nid24.config="as=1 seq=2"
hint.hdaa.0.nid26.config="as=1 seq=1"
hint.hdaa.0.nid27.config="as=1 seq=2"
hint.hdaa.0.nid30.config="as=1 seq=1"

### /etc/sysctl.conf

# Disable real-time channel state notifications to mixers
hw.snd.verbose=0

# Lock the mixer to standard hardware limits (stops volume over-amplification).
# I did various tinkering with this, not sure what's best.
#hw.snd.vpc_0db=1
hw.snd.vpc_0db=75

# Raise the latency slightly to prevent the control panel from causing an underrun.
# I tried various values from 2 to 8 here, not sure what's best.
hw.snd.latency=2

# Stop polling the hardware. 
dev.hdac.0.polling=0

# Smooth out the processing window to eliminate clicks. Tried various values here.
# This was tried earlier, but is not a valid FreeBSD 15 sysctl OID:
# hw.snd.feeder_buffersize=4096
hw.snd.feeder_rate_quality=4

# Allow multiple applications to share the hardware channels natively. In theory,
# these didn't seem to help, but leaving them here just in case.
#hw.snd.maxautovchans=8
#dev.pcm.0.play.vchans=8

# I think this is the default for the realtek audio, but just in case.
dev.pcm.0.play.vchanrate=48000

### ~/.config/pulse/daemon.conf

# Force the exact hardware sample clock used by Intel/Realtek HDA
default-sample-format = s16le
default-sample-rate = 48000
alternate-sample-rate = 48000

# Force stable, non-dynamic resampling parameters
resample-method = speex-float-3
avoid-resampling = yes

### ~/.config/pulse/default.pa

This was the relevant section if I recall.

### Automatically load driver modules depending on the hardware available
#.ifexists module-udev-detect.so
#load-module module-udev-detect
#.else
### Use the static hardware detection module (for systems that lack udev support)
#load-module module-detect
#.endif
### FreeBSD devd is used in addition to static detection (only handles hotplug)
#.ifexists module-devd-detect.so
#load-module module-devd-detect
#.endif
### Static clean hardware definition to prevent mixer panel dropouts
load-module module-oss device="/dev/dsp0" sink_name=output source_name=input fragments=8 fragment_size=1024

# Firefox

Go into about:config and set:

# Force Firefox to use your sound instead of creating its own channel. The
# problem I couldn't solve was that it kept setting the volume to 100%, and
# resetting it back to 100% every time there was a commercial in youtube or wherever.
media.cubeb.backend = "oss"

# This was my original attempt to control the max volume issue above, but
# I'm not strictly sure it's necessary or working. But, I've left it for now.
media.volume_scale = 0.3

### Things That Failed

cpu_microcode caused a freeze during startup. I had to learn what gpi
was and how to get out of that mess. Ugly.

hint.hdac.0.config="ovref100" - This was originally an attempt to solve
quiet headphones, but it just caused scratchy sound. And the headphone
issue was mainly solved by adjustable volume on the headphones themselves.

### General Weirdness

During installation when it asks for regdomain, do not accept the default.
Instead, explicitly set the values, even if the default is correct.

What I was experiencing was that it could connect from the USB stick, but as
soon as I booted normally it could no longer connect. I suspect that by not
accepting the default, it explicitly writes some information to a config file
somewhere, and that got it working for me.
